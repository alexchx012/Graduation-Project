## 完成内容

### 1. 联通 ROS2（WSL2 Ubuntu Humble）与 Win11 训练环境（Isaac Sim 5.1.0）通信

* 定位并解决 WSL2 内 `rclpy` 导入失败问题：原因是 conda 的 Python 3.13 覆盖了系统 Python，导致 ROS2 Humble（Python 3.10 编译）二进制扩展不匹配；改用 `/usr/bin/python3` 并关闭 conda auto base 后恢复正常。
* WSL2 内自测 ROS2 正常：通过 `demo_nodes_cpp` talker + `ros2 topic echo /chatter` 验证 DDS 通信正常（注意先启动 publisher，再启动 echo，必要时显式指定类型）。
* Win11 Isaac Sim 侧启用扩展：开启 `ROS 2 BRIDGE (isaacsim.ros2.bridge)`。
* Isaac Sim → WSL2 方向验证：

  * 在 Isaac 中生成 ROS2 Generic Publisher Graph（发布 `std_msgs/msg/Float32` 到 `/topic`），Play 后 WSL2 使用 `ros2 topic info -v /topic`、`ros2 topic hz /topic`、`ros2 topic echo /topic std_msgs/msg/Float32` 验证能稳定接收（~60Hz）。
* WSL2 → Isaac Sim 方向验证：

  * WSL2 发布 `/wsl_str`（`std_msgs/msg/String`），Isaac Sim Action Graph 中使用 `ROS2 Subscriber` 节点订阅，WSL2 侧 `ros2 topic info -v /wsl_str` 显示 `Subscription count: 1`，确认 Isaac 已成功订阅。

## 遇到的问题

* Isaac Sim Script Editor / Kit Python 直接 `import rclpy` 报 `ModuleNotFoundError`：Kit Python 非 ROS2 环境，不能用“pip 装 rclpy”方式解决，需走 Isaac ROS2 Bridge/OmniGraph 路径。
* WSL2 内 `import rclpy` 失败：conda Python 3.13 与 ROS2 Humble 的 Python 3.10 ABI 不匹配，导致 `_rclpy_pybind11` 扩展缺失；改用系统 Python 并关闭 conda auto base 解决。
* `ros2 topic echo` 提示无法确定类型/未发布：由于启动顺序或 discovery 延迟；通过先启动 publisher、显式指定 message type 规避。
* Isaac 菜单模板缺少 Subscriber：通过 Action Graph 手动添加 `ROS2 Subscriber` 节点完成订阅。
