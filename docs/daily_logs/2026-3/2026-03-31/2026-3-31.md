### 1. 实现 Reward-Terrain Trap 三项修复

基于 3/30 的根因分析（MORL v2 训练陷入"站着不动"陷阱），实现了三项代码修复。

#### 修复 A：MORL 权重 Curriculum

让 MORL secondary rewards 在训练初期为零（纯 scaffold 学走路），然后线性增长到完整权重。

**改动文件：**

| 文件                                              | 改动                                                                                                            |
| ------------------------------------------------- | --------------------------------------------------------------------------------------------------------------- |
| `src/go1-ros2-test/envs/mdp/morl_rewards_v2.py` | 添加 `_morl_curriculum_scale()` 函数，所有 4 个 MORL v2 reward 函数乘以该 scale                               |
| `src/go1-ros2-test/envs/morl_env_cfg_v2.py`     | 添加 `morl_curriculum_warmup_steps` / `morl_curriculum_ramp_steps` 配置（默认 0，向后兼容）                 |
| `scripts/go1-ros2-test/train.py`                | 添加 `--morl_curriculum_warmup` / `--morl_curriculum_ramp` CLI 参数（单位：迭代数），自动转换为 env step 数 |

**机制：**

- 用 `env.common_step_counter` 作为时间基准
- warmup 阶段（默认 300 iter）：MORL rewards 全部为 0，只有 scaffold 驱动训练
- ramp 阶段（默认 300 iter）：MORL rewards 线性从 0 增长到 1
- 之后：完整 MORL 权重

#### 修复 B：去掉 ang_vel_z 免费午餐

原 `repair_forward_v1` profile 设置 `ang_vel_z=(0.0, 0.0)`，导致 `track_ang_vel_z_exp` 白送 0.72/step 给站着不动的 policy。

**改动文件：**

| 文件                                  | 改动                                                          |
| ------------------------------------- | ------------------------------------------------------------- |
| `scripts/go1-ros2-test/morl_cli.py` | 新增 `repair_forward_v2` profile：`ang_vel_z=(-0.5, 0.5)` |

#### 修复 C：降低 clip_param

原 `DEFAULT_CLIP_PARAM=0.3`（比 baseline 0.2 大 50%），允许过大的 policy 变化，加剧第一次 PPO update 的破坏性。

**改动文件：**

| 文件                                           | 改动                                   |
| ---------------------------------------------- | -------------------------------------- |
| `scripts/phase_morl/run_morl_train_sweep.py` | `DEFAULT_CLIP_PARAM` 从 0.3 改为 0.2 |
