### 2. Energy=0 根因排查与修复

#### 根因：与 speed 问题完全相同——指数核 scale 相对于物理量级过大

**[Fact]** energy 奖励函数：
```python
power = Σ|joint_vel × applied_torque|  # 12 关节总机械功率
return exp(-scale * power)             # scale=1.0
```

**[Fact]** P2（能效优先 weight=0.7）报告 energy=0.0001：
- raw_reward = 0.0001 / 0.7 ≈ 0.000143
- exp(-1.0 × power) = 0.000143 → power = -ln(0.000143) ≈ 8.85W

**[Inference + Evidence]** Go1 功率范围估算：
- 站立维持姿态：~5-10W（已产生 exp(-8.85) ≈ 0.000143 的微弱信号）
- 高效行走：~50W → exp(-50) ≈ 1.9e-22 → **完全死区**
- 正常行走：~150W → exp(-150) ≈ 7.2e-66 → **完全死区**

**[Inference]** 因果链与 speed 问题完全对称：
```
训练初期 policy 随机 → 关节抖动产生功率 ~50-200W
  → reward = exp(-1.0 × 100) ≈ 3.7e-44
  → PPO advantage ≈ 0 → 无有效梯度
  → energy 奖励自始至终为 ≈ 0
```

对比 smoothness（正常工作）：
- P3 报告 smooth=0.6447, weight=0.7, raw=0.921
- exp(-0.01 × action_rate_sq) = 0.921 → 信号健康
- **原因：smoothness 的 scale=0.01 已经正确匹配了 action_rate_sq 的量级**

#### 修复方案：scale 1.0 → 0.005
