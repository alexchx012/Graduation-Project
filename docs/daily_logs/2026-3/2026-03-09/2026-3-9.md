#### 变体 A 分析（track_lin_vel_xy_exp 1.5→3.5）— ✅ 显著改善

- 稳态误差从 0.153 降至 **0.068**（降幅 56%），假设验证成功
- `stable_ratio` 从 0.824 升至 **0.950**（几乎所有步都在 0.15 误差以内）
- `mean_vx_meas=1.007` 几乎完美跟踪 `cmd_vx≈0.96`
- 无 base_contact（无翻倒），说明更高的跟踪权重**没有**损害稳定性
- 结论：**提升速度跟踪权重是有效的调优手段**，且在当前权重区间内无明显副作用

#### 变体 B 分析（action_rate_l2 -0.01→-0.05）— ❌ 严重退化

- `mean_vx_meas=0.55`（目标 1.0），机器人几乎只跑到一半速度
- `mean_vx_abs_err=0.462`，是 baseline 的 3 倍
- `stable_ratio=0.179`，仅 18% 的步骤在误差阈值内
- 出现 base_contact=0.007（有翻倒），baseline 和变体 A 均为 0
- 根因：`action_rate_l2` 权重 -0.05 过强（5 倍于 baseline 的 -0.01），动作平滑惩罚压制了速度跟踪激励，agent 学会了"不动"而非"平滑地动"
- 结论：**action_rate_l2 的敏感度远高于 track_lin_vel_xy_exp**，×5 的调整已然过激

#### 3.8.2 分析：地形难度梯度

**Slope 10° → 20°**：

- `abs_err` 从 0.188 升至 0.198（+5.3%），退化幅度很小
- `mean_vx_meas` 从 0.947 微升至 0.950，几乎无差异
- `p95_vx_abs_err` 从 0.207 跃升至 0.800（+286%），说明 20° 坡地偶尔出现大误差
- 结论：**Rough baseline 对坡地泛化良好**，10°→20° 的平均性能几乎不退化，但尾部误差显著增加

**Stairs 10cm → 15cm (OOD)**：

- `abs_err` 从 0.193 到 0.193，**几乎无差异**
- `stable_ratio` 从 0.927 到 0.945（Stairs15 的 thresh=0.30 更宽，不可直接比较）
- `base_contact_rate` 从 0.000006 升至 0.000017（均极低）
- 结论：**15cm 台阶超出训练分布（上限 0.1m），但性能未显著退化**，Rough baseline 的 OOD 泛化能力超出预期

#### 3.8.3 分析：坡地 vs 台阶

在相同 `stable_err_thresh=0.20` 下比较 Slope10/20 与 Stairs10：

|                  | Slope10 | Slope20 | Stairs10 |
| ---------------- | ------- | ------- | -------- |
| `abs_err`      | 0.188   | 0.198   | 0.193    |
| `p95_abs_err`  | 0.207   | 0.800   | 0.800    |
| `base_contact` | 0.0     | 0.0     | 0.000006 |

- 平均误差三者接近（0.188-0.198），**坡地和台阶对 Go1 的平均挑战程度相当**
- 尾部误差（p95）在 Slope20 和 Stairs10 上都约 0.80，均显著高于 Slope10 的 0.207
- 台阶出现极少量翻倒（0.000006），坡地零翻倒 — **台阶地形对稳定性略有影响**

#### 3.8.4 分析：与 Flat baseline 对比

|             | Flat baseline  | Rough baseline（混合） | 专项地形平均 |
| ----------- | -------------- | ---------------------- | ------------ |
| `abs_err` | 0.0677 (变体A) | 0.0798                 | 0.193        |

专项地形的 `abs_err` 约为 Flat baseline 的 2.9 倍。主要原因：

- 地形扰动导致实际速度波动更大
- ROS2 command 与实际机体运动之间的延迟在粗糙地形上被放大
- 但所有组均在宽松标准内通过，说明 Rough baseline 的核心行走能力是充分的

#### 3.8.5 分析：OOD 表现（Stairs15）

Stairs15 是本阶段最关键的发现：

- 训练地形 `grid_height_range` 上限 = 0.1m，15cm 台阶超出 50%
- 但 `abs_err=0.193`、`stable_ratio=0.945`、`base_contact=0.000017`，**表现与训练分布内的 Stairs10 几乎一致**
- 可能原因：训练时 `pyramid_stairs` 的 `step_height_range=(0.05, 0.23)` 上限为 0.23m，**15cm 实际上在 `pyramid_stairs` 训练范围内**；超出范围的是 `boxes.grid_height_range` 而非 `pyramid_stairs`
- 修正结论：**Stairs15 可能并非真正 OOD**，需要在第 6 章中准确区分 `boxes` 和 `pyramid_stairs` 的训练范围

#### 3.8.6 分析：Reward 项差异（不同地形下的表现）

不同地形类型对各 reward 项的影响方式不同，以下逐项分析：

**1. `track_lin_vel_xy_exp`（速度跟踪，权重 1.5）**

- 所有 4 种专项地形的 `mean_vx_abs_err` 均在 0.188–0.198 范围内，远高于 Rough 混合地形的 0.080（+135%~148%）
- 坡地上：重力分量投影到前进方向，上坡减速/下坡加速，导致速度跟踪误差增大
- 台阶上：足部碰撞和悬空导致瞬时速度波动，平均误差与坡地相当
- `p95_vx_abs_err` 在 Slope20/Stairs10/Stairs15 上均约 0.80，而 Slope10 仅 0.207 — 表明高难度地形的**尾部误差**显著增加，reward 函数的指数形式 $\exp(-\|v-v_{cmd}\|^2/\sigma^2)$ 会放大这些尾部事件的惩罚

**2. `flat_orientation_l2`（姿态水平惩罚，Rough baseline 权重 0.0 — 已关闭）**

- 该项在 Rough baseline 中权重为 0.0（关闭），是坡地上表现良好的**关键设计决策**
- 如果启用（如 Flat baseline 的 -2.5），坡地上机体必然倾斜（10° 坡约 0.17 rad，20° 坡约 0.35 rad），将持续产生大量惩罚
- 这会迫使 agent 学习"抗拒倾斜"的保守策略，反而降低行走能力
- 关闭此项使 agent 可以自由适应坡地倾斜，专注于速度跟踪

**3. `feet_stumble`（绊脚检测，权重 -1.0）**

- 该项检测足端与地面的非预期接触（碰撞力超过阈值时触发惩罚）
- 台阶地形上碰撞检测触发频率预期更高：台阶边缘产生碰撞冲击，`feet_stumble` 惩罚更频繁
- 间接证据：Stairs10 的 `base_contact_rate=0.000006`、Stairs15 为 `0.000017`，而坡地均为 0.0 — 台阶上偶发翻倒说明足部碰撞力更大
- 在训练中，`feet_stumble` 惩罚引导 agent 学习抬腿过台阶的策略，减少足部拖地

**4. `feet_air_time`（滞空时间奖励，权重 0.25）**

- 该项鼓励足部有足够的离地时间（促进自然步态节奏）
- 台阶地形对步态节奏产生扰动：上/下台阶时足部提前或延迟着地，改变 air_time 分布
- 坡地上步态节奏受影响较小（连续斜面 vs 离散台阶）
- 训练中该项与 `feet_stumble` 协同作用：鼓励抬腿（air_time ↑）+ 惩罚碰撞（stumble ↓）= 更清晰的台阶越障策略

**总结**：坡地主要通过重力分量影响 `track_lin_vel_xy_exp`；台阶主要通过碰撞冲击影响 `feet_stumble` 和 `feet_air_time`。`flat_orientation_l2=0.0` 是 Rough baseline 适应坡地的前提条件。

## 遇到的问题

### 1. target_vx 不一致导致正式评估全部失败（已修复）

- 根因：ROS2 publisher vx=0.8，eval.py 默认 target_vx=1.0，硬编码容差 0.05
- 修复：`--target_vx 0.8` + 新增 `--pass_cmd_vx_tol` 参数替换硬编码
- 影响：需重跑 4 组正式评估

### 2. Slope10 通过标准过严（已修复）

- 根因：Slope10 使用 `abs_err≤0.15, stable_ratio≥0.85, thresh=0.15`，比 Rough 混合地形 baseline 还严
- 修复：放宽至与 Slope20 一致（`abs_err≤0.25, stable_ratio≥0.70, thresh=0.20`）

### 3. Stairs15 首次运行命令拼写错误（已修复）

- 根因：`--strict_passf`（多了一个 f）
- 修复：改为 `--strict_pass`，重跑成功
