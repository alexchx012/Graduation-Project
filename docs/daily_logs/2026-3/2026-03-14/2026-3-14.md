### 1. 交叉评估脚本改造

将 `scripts/phase4-ppo-dr/run_phase4_eval_all.py` 的 `cross` 模式从“固定使用 seed42”改成了“自动读取 standard 评估结果并选择 best seed”。

本次改动的目标：

- 让 4.7.2 与 `phase4_plan.md` 保持一致
- 交叉评估不再依赖手工判断 best seed
- 在 standard 评估尚未生成时保留保守回退路径，避免脚本完全不可用

### 2. best seed 自动选择逻辑

新增 helper：

- `_extract_seed_from_stem()`：从 `baseline_seed42`、`dr_mass_seed43` 这类文件名中提取 seed
- `_load_json()`：用 `utf-8-sig` 安全读取评估 JSON，兼容 PowerShell 生成的 BOM 文件
- `_pick_best_standard_seed()`：从 `logs/eval/phase4_formal/<category>/` 中挑选 best seed

判定规则：

1. `pass=True` 优先
2. `mean_vx_abs_err` 越低越好
3. `stable_ratio` 越高越好
4. `mean_base_contact_rate` 越低越好
5. 若仍然相同，则选更小的 seed 作为稳定 tie-break

实现结果：

- `cross` 模式现在会先读取 standard 评估 JSON
- baseline 从 `logs/eval/phase4_formal/baseline/baseline_seed*.json` 中选 best seed
- `dr_friction` / `dr_mass` / `dr_push` 从 `logs/eval/phase4_formal/dr/<exp>_seed*.json` 中选 best seed
- 如果对应 standard JSON 尚未生成，则自动回退到 `seed42`

### 4. standard 评估失败根因定位

继续检查 `logs/sweep/phase4_eval/phase4_eval_standard_20260314_101738.log` 后，确认这不是 ROS2 publisher 故障，也不是 checkpoint 实际缺失。

定位结果：

- ROS2 publisher 已正常启动，说明评估链路不是在通信侧先失败
- 前 9 个任务全部以同一个错误退出：`FileNotFoundError: Unable to find the file: model_1499.pt`
- 训练目录内实际存在 `model_1499.pt`
- 根因在 `scripts/go1-ros2-test/eval.py`：只要传了 `--checkpoint model_1499.pt`，脚本就会把它当成独立文件路径解析，直接忽略 `--load_run`

这意味着批量评估脚本传入的：

```powershell
--load_run D:\...\2026-03-08_16-46-27_baseline_rough_ros2cmd --checkpoint model_1499.pt
```

会被 `eval.py` 误解成“去当前工作目录找 `model_1499.pt`”，从而导致 standard 评估在前 9 个任务上全部失败。

### 5. eval.py checkpoint 解析修复

按“方案 2”修复了 `scripts/go1-ros2-test/eval.py`：

  - 绝对 run 目录路径

- 绝对 checkpoint 路径的旧用法保持不变

#### 6.2 按计划问题逐项分析

1. **收敛速度**直接比较 `best_ppo_dr_analysis.json` 中 low/high 变体的平均收敛 iter，`clip_param` 的影响最大：`PPO-Clip-Low = 230.33` vs `PPO-Clip-High = 464.33`，差值 `234.00`；`learning_rate` low/high 差值为 `86.33`；`entropy_coef` low/high 差值为 `53.67`。因此，按现有训练曲线证据，**Clip 是对收敛速度影响最大的 PPO 因子**。
2. **最终性能**在 standard 评估的 3-seed 均值上，**`PPO-Clip-High` 的 `mean_vx_abs_err` 最低，为 `0.1205 ± 0.0899`**，是当前全部 Phase 4 配置中最好的标准环境结果。相比 baseline 的 `0.4051 ± 0.5143`，均值差为 `-0.2846`。按 `mean_vx_abs_err` 计算的 Cohen's d 为 `-0.771`，方向上优于 baseline，但**未达到计划中“效应量 > 0.8”的强结论阈值**。直接文件证据支持“`PPO-Clip-High` 是当前 best standard-eval 配置”，但**不支持把这件事写成强统计显著结论**。
3. **稳定性**如果只看原始方差，`DR-Friction` 的 `mean_vx_abs_err` 标准差最小（`0.000042`），但这是因为 **3 个 seed 都以几乎相同的方式失败**，`stable_ratio=0` 且 `pass=0/3`，不具备“可用稳定性”意义。在可用的 PPO 配置里，**`PPO-Clip-High` 的 `mean_vx_abs_err` 标准差最小（`0.0899`），且通过数 `2/3`**，因此可以作为当前 **最稳定的可用 PPO 配置**。
4. **DR 泛化**standard 环境中，3 组 DR 全部 `0/3` 通过，且 `mean_vx_abs_err` 都接近 `1.0`、`stable_ratio=0`。cross 环境中，`cross_baseline` 使用 `baseline_seed44`，结果为 `abs_err=0.070880`、`stable_ratio=0.990741`、`pass=true`；而 `cross_dr_friction` / `cross_dr_mass` / `cross_dr_push` 分别为 `1.000624` / `0.997465` / `0.999148`，`stable_ratio` 全为 `0`，全部失败。**直接证据不支持“DR 提升泛化能力”；现有结果反而显示 3 组 DR 在标准环境与抗扰动环境下均出现严重跟踪失败。**
5. **安全性**standard 与 cross 中，3 组 DR 的 `mean_base_contact_rate` 都是 `0.0`，但同时 `mean_vx_meas` 也接近 `0.0`、`stable_ratio=0.0`。因此，现有数据能直接说明的是：**DR 失败并未表现为 base contact 飙升**；但 `[Inference]` 由于这些 run 同时基本不前进，`base_contact_rate=0.0` 不能直接解释为“更安全”，更像是**跟踪任务已经塌缩到近似静止状态**。
6. **PPO vs DR**PPO 和 DR 的方向完全相反：

   - best PPO（standard 视角）=`PPO-Clip-High`，`abs_err=0.1205 ± 0.0899`，`pass=2/3`
   - 3 组 DR 全部 `0/3`，且 `abs_err≈1.0`
     同时，训练曲线文件 `best_ppo_dr_analysis.json` 又显示 DR 组拥有更高的训练 reward（如 `DR-Mass` reward group mean=`42.5293`，收敛 `206.33`）。
     这形成了直接证据链：**训练 reward 排名与最终速度跟踪评估排名不一致**。
     `[Inference]` 对当前 Phase 4 而言，**standard eval 比训练 reward 更能代表最终可用性**。
7. **组合效应**
   组合实验 `Combo (clip_high + dr_mass)` 的 standard 结果为 `1.0043 ± 0.0033`，`pass=0/3`。
   它不仅远差于 `PPO-Clip-High`（`0.1205 ± 0.0899`，`2/3`），也略差于 `DR-Mass`（`0.9995 ± 0.0018`，`0/3`）。
   因此，**组合效应没有带来任何增益；直接结果是“组合后比 PPO 单独更差，并且没有修复 DR-Mass 的失败模式”**。

#### 6.3 当前可支持的结论

- **按 standard eval 结果，当前 best PPO 是 `PPO-Clip-High`，不是训练曲线阶段初判的 `PPO-Clip-Low`**
- **按 standard eval 结果，3 组 DR 全部失败，不能推荐任何 DR 变体作为最终配置**
- **当前 combo (`clip_high + dr_mass`) 明确失败，不能作为最终推荐组合**
- **截至 4.8 完成时，最强的可证据化结论是：`PPO-Clip-High` 是当前唯一显著优于其余 Phase 4 候选的可用配置；DR 与 combo 均不成立**

#### 6.4 统计严谨性声明

- n=3，统计功效非常有限；以下判断以 **方向、量级、通过数、效应量** 为主，不使用 p 值做强结论
- baseline 与多组 PPO 的标准差很大，说明存在明显 seed 级离群现象；因此所有“优于 baseline”的说法都需要保留方差说明
- `DR` / `combo` 的失败模式高度一致（`abs_err≈1.0`, `stable_ratio=0`），这类结论不依赖边际统计显著性，属于直接观测结果
