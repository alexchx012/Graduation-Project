## 遇到的问题

### 1. ppo_clip_low_seed43 训练崩溃：策略标准差变负

- **失败实验**：`ppo_clip_low_seed43`（clip_param=0.1, seed=43）
- **错误信息**：`RuntimeError: normal expects all elements of std >= 0.0`
- **错误位置**：`torch.normal(self.loc.expand(shape), self.scale.expand(shape))`
- **运行时长**：约 1.5h（ROS2 Watchdog 全程正常）
- **错误类别**：**非瞬态错误**，与 Isaac Sim `torch.jit.load` 瞬态崩溃不同
- **原因分析**：

  - PPO 策略网络的标准差（std）参数在训练过程中变成了负值或 NaN
  - 采样动作时调用 `torch.normal()` 要求 std ≥ 0，触发 RuntimeError
  - `clip_param=0.1` 是非常保守的裁剪范围，可能导致某些 seed 下梯度更新不稳定
  - 这是 PPO 数值不稳定问题，不是环境或框架 bug
- **风险评估**：

  - 如果 `ppo_clip_low` 的其他两个 seed（42, 44）正常完成 → 大概率是 seed=43 的随机数值不稳定，重跑有望成功
  - 如果 3 个 seed 里有 2 个以上失败 → `clip_param=0.1` 本身对 Go1 不适用，属于系统性失败，重跑无意义
- **解决方法**：等 18 次训练全部结束后，单独重训失败实验

  **方法 A：手动运行单条命令**

  ```powershell
  conda activate env_isaaclab

  python scripts/go1-ros2-test/train.py ^
    --task Isaac-Velocity-Rough-Unitree-Go1-ROS2Cmd-v0 ^
    --num_envs 4096 --max_iterations 1500 --seed 43 --headless ^
    --disable_ros2_tracking_tune ^
    --run_name ppo_clip_low_seed43 ^
    --clip_param 0.1
  ```

  注意：运行前需确保 WSL ROS2 publisher 正在运行。

  **方法 B：用 `--resume` 重跑失败项**

  1. 查看 session 文件：`cat logs/sweep/phase4_train/session_ppo.json`
  2. 删除 JSON 中 `ppo_clip_low_seed43` 的记录（状态为 FAIL 的条目）
  3. 重新运行：
     ```powershell
     python scripts/phase4-ppo-dr/run_phase4_train_sweep.py --mode ppo --resume
     ```
  4. `--resume` 会跳过所有已完成实验，只重跑被删掉的那条
