## 遇到的问题

### 1. `-SkipRos2` 只在 ps1 层生效，未传递到 Python 评估脚本

问题现象：

```powershell
.\scripts\phase_morl\run_morl_eval_all.ps1 -SkipRos2
```

执行后仍然报 ROS2 bridge 启动超时：

```text
[ROS2 Bridge] TIMEOUT after 15.0s — no message received on '/go1/cmd_vel'
RuntimeError: ROS2 bridge: No message received on '/go1/cmd_vel' within 15.0s.
```

根因：

1. `run_morl_eval_all.ps1` 的 `-SkipRos2` 只控制 WSL publisher 启停，没有把 flag 传给 `run_morl_eval.py`
2. `run_morl_eval.py` 原先没有 `--skip_ros2` 参数，task 命中 `ROS2_MORL_TASK_IDS` 后会强制启动 ROS2 bridge

处理结果：

- `run_morl_eval.py` 新增 `--skip_ros2`
- Python 侧补上 ROS2 bridge 初始化、zero-command 检查和 Windows ROS2 bootstrap 的跳过条件
- `run_morl_eval_all.ps1` 补上 `--skip_ros2` 参数透传

### 2. 首轮全量评估在 `P2 seed42` 处出现 Go1 actuator asset 读取异常

首轮执行在 `2026-03-22_05-36-51_morl_p2_seed42` 处中断，错误为：

```text
RuntimeError: PytorchStreamReader failed reading zip archive: failed finding central directory
```

当前判断：

- 错误发生在 `env = gym.make(...)` 的环境初始化阶段
- 更像是 Go1 actuator asset 的一次性读取/缓存抖动，而不是 `model_1499.pt` checkpoint 本身损坏
- 该 run 重试后正常完成，20 个当前有效 run 最终已全部评估完毕
