### 1. v2 MORL 训练失败根因深挖（只读分析，无新训练）

#### 背景

v2 cold 600 iter 和 v2 warm 600 iter 训练后机器人都不走。baseline checkpoint 直接放进 v2 play 能走（mean_vx=0.965）。

#### 深挖发现

**发现 1：model_0 已包含第一次 PPO update**

- 比较 baseline model_1499.pt（init_checkpoint）与 warm model_0.pt 的 policy weights：max diff ~0.004
- 证实 rsl_rl 的 learn() 循环是：rollout → PPO update → save
- 第一次 PPO update 步长约是后续平均步长的 20 倍（fresh Adam optimizer 无动量估计）

**发现 2：iter 599 的 policy 在站着不动**

- morl_energy RAW=0.918 → 功率仅 17W（走路需 200W+）
- morl_smooth RAW=0.940 → 几乎不改变 action
- morl_stable RAW=0.912 → 几乎不摇晃
- track_lin_vel_xy RAW=0.087 → 极差的速度跟踪
- episode_length=995 → 跑满全集，没被 terminate

**发现 3：agent.yaml 差异**

- baseline: clip_param=0.2
- warm v2: clip_param=0.3（DEFAULT_CLIP_PARAM in run_morl_train_sweep.py）
- 两者均 disable_ros2_tracking_tune

**发现 4：command profile 的 ang_vel_z=[0,0] 给站着白送奖励**

- track_ang_vel_z_exp RAW=0.960 → 权重 0.75 → 0.72/step
- 站着不转就拿满，不需要任何行动

#### 根因判断：Reward-Terrain Trap

在 rough terrain + episode termination 的设置下，PPO 优化的是总集回报（不是每步回报）：

- 走路但 100 步摔倒：2.25 × 100 = 225
- 站着不动跑满 995 步：0.93 × 995 = 925

PPO 理性选择"站着"。MORL 的 energy/smooth/stable 项（总权重 0.2）额外奖励不动，进一步加深陷阱。

五个加剧因素：

1. MORL 奖励不对称（0.2 奖励不动 vs 0.05 奖励走）
2. ang_vel_z=[0,0] 免费午餐
3. Fresh Adam optimizer 第一步 20 倍大
4. clip_param=0.3（比 baseline 0.2 大 50%）
5. Critic value 失配

#### 之前路线的错误

Pilot A/B/C 把问题归结于 command profile 或 warm-start 方式。真正的 blocker 是 reward-terrain trap。forward-only command profile 甚至加重了问题（ang_vel_z=[0,0] 让站着白拿 track_ang 奖励）。
