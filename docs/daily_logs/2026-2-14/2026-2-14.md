## 完成内容

### 1. WSL ROS2 脚本命令节点（阶段 6）

新增 `scripts/go1-ros2-test/ros2_nodes/go1_cmd_script_node.py`。

- 运行在 WSL Ubuntu-22.04 系统 Python，不使用 conda
- 发布 `geometry_msgs/Twist` 到 `/go1/cmd_vel`，默认 20Hz
- 支持三种 profile：`constant`（恒定）、`sine`（正弦 0.1Hz）、`step`（每 5s 切换）
- 支持 `--duration` 运行时长限制
- 通过 `rclpy.spin()` + `KeyboardInterrupt`/`SystemExit` 异常捕获 + `finally` 块实现优雅退出

### 2. 模型推理桩节点（阶段 7）

新增 `scripts/go1-ros2-test/ros2_nodes/go1_cmd_model_node.py`。

- 订阅 `/go1/obs_flat`（`Float32MultiArray`），发布 `/go1/cmd_vel`（`Twist`）
- 当前 `_infer()` 为桩函数，返回固定 `(vx=0.3, vy=0.0, wz=0.0)`
- 预留 `torch.load(model_path)` 占位
- 观测日志每 100 条打印一次（`_obs_count` 计数器节流），避免高频输出轰炸
- 同样使用 `KeyboardInterrupt` + `finally` 模式优雅退出

### 3. 运行编排脚本（阶段 8）

新增两个脚本：

**`scripts/go1-ros2-test/run/run_ros2_cmd_node.sh`**

- WSL 端启动器，`set -euo pipefail`，自动 `source /opt/ros/humble/setup.bash`，转发所有 CLI 参数

**`scripts/go1-ros2-test/run/run_go1_ros2_train.ps1`**

- Windows 一键编排：后台启动 WSL ROS2 节点 → 等 3s → conda activate env_isaaclab → isaaclab.bat 训练
- `try/finally` 确保 WSL 进程被清理（`pkill -f` + `Kill()`）
- 支持 `-Profile`、`-NumEnvs`、`-MaxIter`、`-Vx`/`-Vy`/`-Wz`、`-Seed` 等参数

### 4. TensorBoard 指标增强（阶段 9.1）
