## 详细说明

### 1. go1_cmd_script_node.py（阶段 6）

WSL 端 ROS2 Twist 命令发布节点，运行在 Ubuntu-22.04 系统 Python（`/usr/bin/python3`），不使用 conda。

**功能：**
- 发布 `geometry_msgs/Twist` 到 `/go1/cmd_vel`（可配置）
- 默认 20Hz 发布频率
- 支持三种命令 profile：
  - `constant`：恒定速度
  - `sine`：正弦波变化（频率 0.1Hz）
  - `step`：每 5 秒在 0 和目标值之间切换

**CLI 参数：**

| 参数 | 默认值 | 说明 |
|-----|-------|------|
| `--topic` | `/go1/cmd_vel` | ROS2 话题名 |
| `--rate` | `20` | 发布频率 Hz |
| `--profile` | `constant` | 命令模式 |
| `--vx` | `0.5` | 前进速度 m/s |
| `--vy` | `0.0` | 侧向速度 m/s |
| `--wz` | `0.0` | 偏航角速度 rad/s |
| `--duration` | `inf` | 运行时长（秒） |

**实现要点：**
- 基于 `rclpy.create_timer(1.0/rate, callback)` 定时发布
- 每条消息打日志 `[t=X.XXs] pub Twist(vx=..., vy=..., wz=...)`
- SIGINT 优雅退出

### 4. run_go1_ros2_train.ps1（阶段 8）

Windows 端一键编排 PowerShell 脚本。

**执行流程：**
1. 解析参数（`$Profile`、`$NumEnvs`、`$MaxIter`、`$Vx` 等）
2. 后台启动 WSL ROS2 脚本节点
3. 等 3 秒稳定
4. `conda activate env_isaaclab` + `isaaclab.bat -p train.py` 训练
5. `finally` 块清理 WSL 进程（`pkill -f go1_cmd_script_node.py`）

**用法示例：**
```powershell
.\scripts\go1-ros2-test\run\run_go1_ros2_train.ps1 -Profile constant -MaxIter 50 -Vx 0.5
.\scripts\go1-ros2-test\run\run_go1_ros2_train.ps1 -Profile sine -MaxIter 100 -Vx 0.3 -Seed 42
```

### 5. ros2_velocity_command.py 指标增强（阶段 9.1）

在 `Ros2VelocityCommand.__init__()` 中新增 3 个 metrics：

```python
self.metrics["cmd_vx"] = torch.zeros(self.num_envs, device=self.device)
self.metrics["cmd_vy"] = torch.zeros(self.num_envs, device=self.device)
self.metrics["cmd_wz"] = torch.zeros(self.num_envs, device=self.device)
```

在 `_update_command()` 末尾每步记录当前命令值：

```python
self.metrics["cmd_vx"][:] = self.vel_command_b[:, 0]
self.metrics["cmd_vy"][:] = self.vel_command_b[:, 1]
self.metrics["cmd_wz"][:] = self.vel_command_b[:, 2]
```

TensorBoard 最终可见 5 个 `Command/` 前缀指标：
- `Command/cmd_vx`、`Command/cmd_vy`、`Command/cmd_wz`
- `Command/cmd_timeout_count`、`Command/cmd_zero_fallback_count`
