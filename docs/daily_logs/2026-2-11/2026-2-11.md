### 2. OmniGraph 方案调试

尝试了多种 OmniGraph 图创建方案，均失败：

| 方案                                         | 结果     | 备注                                 |
| -------------------------------------------- | -------- | ------------------------------------ |
| 路径 `/ActionGraph`                        | 无效     | wrap 失败                            |
| 路径 `/World/ROS2CmdVelGraph`              | 无效     | 错误相同                             |
| 使用 `push` evaluator                      | 无效     | 与 `execution` 错误相同            |
| `og.Controller.create_graph()` 分步创建    | 无效     | 同样 wrap 失败                       |
| 调用 `timeline.play()` 后创建图            | 无效     | 仍然失败                             |
| 多次 `app.update()` 确保 stage 稳定        | 无效     | prim 能创建但 wrap 失败              |
| 创建前 `og.Controller.delete_graph()` 清理 | 部分有效 | 避免重复创建错误，但不解决 wrap 问题 |

**核心错误：**

```
OmniGraphError: Failed to wrap graph in node given {'graph_path': '...', 'evaluator_name': 'execution'}
[Error] [omni.graph.core.plugin] Unable to create prim for graph at /ActionGraph
```

**诊断发现：**

- pre-flight 检查显示 prim 不存在
- 创建后诊断显示 `prim_exists: True, prim_type: 'OmniGraph'`
- 说明 prim 创建成功，但 OmniGraph 内部的 "wrap" 操作失败

### 3. 根因分析与方案转换

**分析官方示例：**

- 查阅 `d:\isaaclab\_isaac_sim\standalone_examples\api\isaacsim.ros2.bridge\clock.py`
- 发现官方示例在 **simulation start 之前** 创建 OmniGraph
- 我们的代码在 `gym.make()` 之后（simulation 已启动）创建，时机不对

**解决方案：改用 rclpy 直接订阅**

参照官方 `subscriber.py` 示例，完全绕过 OmniGraph：

- 使用 Isaac Sim 内置的 rclpy（`isaacsim.ros2.bridge` 扩展提供）
- 在 physics callback 中调用 `rclpy.spin_once(timeout=0.0)` 处理消息
- 用 `threading.Lock` 保护共享数据

## 遇到的问题

### 1. OmniGraph 无法在 simulation 运行后创建

**现象：** `og.Controller.edit()` 创建 prim 成功，但 wrap 失败

**原因：** Isaac Sim 的 OmniGraph 需要在 simulation start 之前创建。Isaac Lab 的 `gym.make()` 在创建 env 时已经启动了 simulation。

**解决：** 改用 rclpy 直接订阅，不依赖 OmniGraph。

### 2. Python 缓存导致代码修改不生效

**现象：** 修改 `twist_subscriber_graph.py` 后运行仍报旧错误

**解决：** 删除 `__pycache__` 目录和 `.pyc` 文件
