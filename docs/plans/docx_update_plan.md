## 3. DOCX 修改安全流程
由于直接修改 `document.xml` 极易损坏格式或破坏 XML 结构，必须遵循以下流程：

### 阶段一：准备与提取
1. 使用 `unpack.py` 解包 `论文初稿.docx` 到 `unpacked_draft/`（若已解包则复用）。
2. 从 `document.xml` 中提取待修改的段落原文，保存为文本用于参考。

### 阶段二：文本起草（安全区）
1. 在普通文本文件中写好第四章 4.2-4.4 节的替换文段（基于 V2 方案事实）。
2. 写好第一章、第二章的补充文段。
3. 重新编排参考文献列表（1-30），建立 [旧编号] -> [新编号] 的映射表。

### 阶段三：XML 替换（高危区）
1. **替换正文内容**：使用 `sed` 或 Python 脚本在 `document.xml` 中精准定位旧段落并替换为新段落的 XML 格式（保留 `<w:p><w:r><w:t>` 结构）。
2. **全局替换引用编号**：根据映射表，在整个 `document.xml` 中替换引用如 `[18]` -> 新编号。注意处理 XML 被打断的情况（如 `[</w:t></w:r><w:r><w:t>18]`）。
3. **替换文末参考文献列表**：将最后的参考文献段落替换为新的 1-30 列表。

### 阶段四：打包与验证
1. 使用 `pack.py` 重新打包为 `论文初稿_v2.docx`。
2. 使用 `validate.py` 检查 XML 合法性。
3. 使用 `pandoc` 转为 markdown 检查文本是否正确更新且无乱码。
